About Me

Greetings! I'm Berkay, a passionate game developer and enthusiast with a love for all kinds of games. Beyond video gaming, I immerse myself in tabletop games, particularly roleplaying games like D&D and Pathfinder, among others. Board games are also a significant part of my gaming repertoire.

My primary goal is to craft enjoyable experiences for players. Throughout my journey, I have actively contributed to various teams, predominantly in the role of a game developer. My focus lies in creating robust systems and implementing them in a clear and maintainable manner. Let's bring great gaming experiences to life!

Published Projects

Match ColorDom!

Company: RoofTop Games

About The Game

Match Colordom! is a puzzle game for mobile users. The goal of the player is to clear the area using the tiles deck. When the player matches the color, the matched tile comes to the hand of the player.

My Contributions

I was the only developer on this project. I planned and implemented the software in the direction of game design that had been made beforehand. The closed and open tiles are determined at the initial phase of each level. This initialization procedure makes the game more designer-friendly.

I also programmed level design tools to randomize and control the tile colors. This made the design process more efficient. Additionaly, I created a program that makes random decks according to the content of the level.

Split 3D!

Company: Two Page Games

About The Game

Split 3D! is a puzzle game for mobile users. The goal of the player is to clear the area by drawing a border around all the tiles of one color. When there is only one color left in the area, the level is completed.

My Contributions

I did all the programming for this game except input handling. Smooth input handling was a tough problem and I consulted my supervisor about that one. We implemented it together.

I used an algorithm that was inspired by GO, an ancient board game, in order to determine whether the line was correct or not.

I used the DreamTeck Spline tool for the first time in this game. Learning, adjusting, and implementing this tool was fun.

The game designer used a tool developed for use in such tile-based games. The first game that I created the tool for was Unwind, another game we developed.

Slide & Deliver

Company: Two Page Games

About The Game

Slide & Deliver is a puzzle game for mobile users. The goal of the player is to create paths for delivering the mail by sliding the islands. When all of the houses get their mail, the level is completed.

My Contributions

I did all the programming for this game.

I used the Breath First Search Algorithm, and it was reusable. Thanks to this situation, the same script could be easily applied if other games needed pathfinding.

I also adjusted the tool that I developed for tile-based games previously for this game's needs.

Parking Space!

Company: Two Page Games

About The Game

Parking Space! is a puzzle game with time constraints. The player should move the vehicle to an empty space; otherwise, BOOM! Vehicles remain in their spots until their waiting time is over and then drive away. When all vehicles are placed and driven away, the level is completed.

My Contributions

I did all the programming for this game.

I applied vertex displacement for the first time. The result of vertex displacement led us to smooth and beautiful feedback.

Unwind!

Company: Two Page Games

About The Game

Unwind! is a puzzle game for mobile devices. The player moves the ball of wool towards the finish line. However, there are several special tiles to utilize or obstacles to overcome.

My Contributions

Lemonade Party

Company: Two Page Games

About The Game

Lemonade Party is an arcade-idle game. The player is the owner of a lemonade store. Players collect fruits, squeeze juice out of them, sell them and recruit more workers to grow their businesses.

My Contributions

Lemonade Party! is the first professional game that I was involved in as a game developer. It was a great start to my exciting journey.

I programmed mainly the store part of the game. In addition to that, I worked with my supervisor on the exploration part of the game.

Finally, we updated the game to add more levels and content. This created a problem for the money. It exceeded the integer boundary; therefore, I implemented a system to hold the money data as an exponential and coefficient.

Jam Projects

KAT vs KAT

Contributors: Ahmet Arif Acıduman, Alara Çiloğlu, Aslı Gençtürk, Berkay Karakaya

About The Game

Kat vs Kat is a multiplayer action-strategy game. Each team consists of four players. Each player will take a button that is connected to an Arduino board. Each member of the team takes one direction to move and one special ability slot. Because we lacked the hardware, we implemented the second team via keyboard. Teams try to collect coins while preventing the other team from collecting coins. There is a time limit; when it finishes, the team that has the most coins wins.

At the end of the jam, Kat vs. Kat was declared the winner.

My Contributions

Arif and I participated as game developers. Alara and Aslı did the art pieces and made design decisions.

We hadn't worked with Ardunio board before this jam. I tried to figure it out firstly to create seperate input handling. After solving that issue, we planned the software system and implemented it.

The time limit was almost 30 hours for this game. We did our best, and we are happy with the outcome.
